Validação de Requisitos — Etapa 1

✅ AUTOAVALIAÇÃO DO DESENVOLVEDOR

🔹 1. SOBRE OS DADOS

Os dados manipulados foram classificados (sensíveis, pessoais, comuns)?
Dados sensíveis estão protegidos (criptografia ou hash)?
Existe justificativa para coleta de cada dado?
Existe definição de onde os dados serão armazenados?
Existe definição de tempo de retenção dos dados?

🔹 2. SOBRE AUTENTICAÇÃO

Senhas são armazenadas com hash seguro (bcrypt/argon2)?
Existe política mínima de senha (tamanho/complexidade)?
Sessões possuem expiração automática?
Existe proteção contra múltiplas tentativas de login?
Existe mecanismo seguro de recuperação de senha?

🔹 3. SOBRE CONTROLE DE ACESSO

Existem perfis de usuário definidos?
O sistema aplica princípio do menor privilégio?
Permissões são validadas no backend?
Existe proteção contra acesso direto por URL/API?

🔹 4. SOBRE ENTRADAS E VALIDAÇÃO

Todas as entradas são validadas no backend?
Existe proteção contra SQL Injection (queries parametrizadas)?
Existe proteção contra XSS?
Existe controle de tamanho/formato de entrada?

🔹 5. SOBRE ARQUITETURA E EXPOSIÇÃO

O sistema utiliza HTTPS obrigatoriamente?
Serviços desnecessários estão desativados?
Integrações externas são validadas e seguras?

🔹 6. SOBRE DEPENDÊNCIAS

Bibliotecas utilizadas são confiáveis e atualizadas?
Existe controle sobre dependências utilizadas?

🔹 7. SOBRE CONFIGURAÇÃO

Credenciais estão fora do código (env)?
Ambiente de produção está sem debug ativo?

🔹 8. SOBRE MONITORAMENTO

O sistema registra eventos críticos (logins, erros)?
Logs evitam exposição de dados sensíveis?

🔹 9. SOBRE RESILIÊNCIA

Existe estratégia de backup definida?
