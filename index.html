Guia de Boas Práticas — Etapa 1

🧩 ETAPA 1: DEFINIR

🎯 Objetivo da Etapa: Estabelecer, antes do início do desenvolvimento, todos os requisitos mínimos de segurança necessários para garantir que o sistema não seja construído com vulnerabilidades estruturais, reduzindo riscos técnicos, operacionais e legais.

🧠 1. COMPREENSÃO DO CONTEXTO

Sistemas expostos à internet devem ser tratados como ambientes hostis por padrão, exigindo maior rigor na aplicação de controles de segurança.

👉 Boa prática: Nunca iniciar o desenvolvimento sem compreender o nível de criticidade.

🧠 2. TRATAMENTO DE DADOS

Identificar e classificar todos os dados que serão manipulados, com atenção especial aos dados sensíveis para aplicar proteção adequada.

👉 Boa prática: Dados sensíveis nunca devem ser armazenados sem proteção adequada.

🧠 3. AUTENTICAÇÃO

Estabelecer políticas de senha, complexidade e recuperação segura. Senhas devem usar hash forte e sessões devem expirar.

👉 Boa prática: A autenticação deve ser projetada antes da implementação.

🧠 4. CONTROLE DE ACESSO

Definir perfis e permissões. Validar obrigatoriamente no backend, não apenas ocultando menus na interface.

👉 Boa prática: Nunca confiar no frontend para controle de permissões.

📘 Tutorial Completo: Instalação do SonarQube

🖥️ PARTE 1 — INSTALAÇÃO NO WINDOWS (PRINCIPAL)

🔹 1. Instalar o Java (JDK 17)

O SonarQube NÃO funciona sem Java.

Passos:

  • Baixe o JDK 17 (recomendo): Eclipse Temurin
  • Instale normalmente (Next → Next → Finish)

✔️ Verificar instalação

Abra o Prompt de Comando (CMD):

java -version

👉 Deve aparecer algo como: openjdk version "17.x.x"

❌ Se der erro → Java não foi configurado no PATH.

🔹 2. Baixar o SonarQube

Acesse: Link oficial e baixe a versão Community (gratuita).

Extraia o ZIP para uma pasta, por exemplo: C:\sonarqube

🔹 3. Estrutura do projeto

Dentro da pasta você verá algo como:

bin/
conf/
data/
logs/
extensions/

🔹 4. Iniciar o SonarQube

Vá até: bin\windows-x86-64

Execute: StartSonar.bat

👉 Vai abrir um terminal com logs.

🔹 5. Aguardar inicialização

Você verá mensagens como: SonarQube is up

📸 [Imagem pendente: print do cmd com “started” ou logs finais]

🔹 6. Acessar no navegador

Abra: http://localhost:9000

📸 [Imagem pendente: print do navegador com tela de login]

Login padrão:

  • user: admin
  • password: admin
📸 [Imagem pendente: print do dashboard inicial]

🔹 7. Trocar senha (obrigatório)

O sistema vai pedir para trocar a senha na primeira vez.

⚠️ Problemas comuns no Windows

❌ Porta 9000 ocupada

Erro: Port already in use

✔️ Solução:

netstat -ano | findstr 9000
📸 [Imagem pendente: print do cmd]

Depois finalize o processo:

taskkill /PID XXXX /F
❌ Java não reconhecido

Erro: java is not recognized

✔️ Solução:

  • Configurar variável de ambiente JAVA_HOME
  • Adicionar ao PATH
